A Big Data Machine Learning as a Service:

  • A scalable Machine Learning as a Service (MLaaS) with Apache Spark and Hadoop.
    • Enable data owner to apply machine learning technology with few clicks
    • Allow flexible data sources: "LIBSVM", "CSV" or "n-gram pattern" text formats
    • Visualized ML pipeline UI that guides user to generate model(s), optimize the performance and predict/inference unknown samples
    • RESTful APIs enable ML pipeline automation and integration with external applications or services
    • Enabled plugin-able ML pipline for custom feature engineering
  • Machine Learning Libraries and Algorithms:
    • Supervised and Unsupervised learnings: classification and clustering
    • Web UI to tune hyperparameters with grid search and cross validation
    • Allow one dataset to exploit multiple training options
    • Deep Neural Network:
      • Supported Theano or Tensorflow backend by Keras
      • CNN/DC, Convolution Neural Network/Dense Connected Neural Network: for image or LIBSVM data
      • LSTM/RNN, Long Short-Term Memory/Recurrent Neural Network: for time series/sequential data
    • Spark MLlib:
      • Classification: Linear SVM and Logistic Regression etc.
      • Clustering: Kmeans
    • SciKit-Learn:
      • Classification: Linear SVM, Logistic Regression and Perception etc.
      • Clustering: Kmeans
    • Ensemble classifier:
      • Allow selectable multiple classifiers from different datasets
      • Fast parallel processing
  • Featuring Tool for Linear Model:
    • Feature list: for model and samples
    • Feature exclusion: to exclusion selected feature for re-training
  • Data format "LIBSVM":
    • Input pre-featured data as "LIBSVM" format: (not for LSTM/RNN)
    • Each row for a sample
    • Format for a data row:<hash/text> <label> <index1>:<value1> <index2>:<value2> …\n
    • All data is numerical, except the first column "hash/text"
    • Each feature represented by <indexN>:<valueN>
    • Meta Data: (optional)
      • Label list:<label name(string)>:<label(number)>
      • Feature list:<feature description(string)> : <indexN(number)>
  • Data format "N-gram pattern": e.g. for API call log file
    • Input raw data as "N-gram pattern" text format with the Regular Expression pattern string specified
    • Each data row for one sample. Allow "time sequence" data. e.g. execution log etc
    • Each observation (string) separated by a "tab" char, "\t"
    • Format for a data row example:<label>\t<md5>\t<date>\t <string1 for pattern match>\t<string2 for pattern match>\t …\n
    • Pattern string example: '^I/AndroidATD\([ ]*[\d]+\): \[API\] (.*) \[.*'
      • substring(s) will be extracted as part of the feature in n-gram (e.g. highlighted in parenthesis)
    • Extracted string will be hashed and n-gram-ized to the "LIBSVM" format
    • Meta Data: (generated)
      • Label list:<label name(string)>:<label(number)>
      • Feature list:<indexN(number)> : <feature description(string)>
  • Data format "CSV": i.e. Comma separated text file
    • Each data row for one sample
    • Header/Column title at first row is required
    • Each observation (string) separated by a comma char, ","; Each column will become one feature
    • Format for header+data rows example: "label","h1","h2"…,"hN"\n "l1","o1","o2"…,"oN"\n ... "ln","on1","on2"…,"onN"\n
    • For web UI, custom "Parameter JSON" example input: {"label_index": "0", "has_header": "True", "custom": "csv"}
    • A hash will be generated for each data row/sample; Data will be converted to LIBSVM and saved in HDFS
    • Meta Data: (generated)
      • Label list:<label name(string)>:<label(number)>
      • Feature list:<indexN(number)> : <feature description(string)>
