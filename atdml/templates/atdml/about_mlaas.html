A Big Data Machine Learning as a Service:

  • A scalable Machine Learning as a Service (MLaaS) with Apache Spark and Hadoop.
    • Enable data owner of all skill levels to apply machine learning technology
    • Allow flexible data sources: "LIBSVM" or "n-gram pattern" formats
    • Visualized ML pipeline UI that guides user to generate model(s), optimize the performance and predict unknown samples
    • RESTful APIs enable ML pipeline automation and integration with external applications or services
  • Machine Learning Libraries and Algorithms:
    • Supervised and Unsupervised learnings: classification and clustering
    • Customizable ML parameter user interface with grid search and cross validation
    • Allow one dataset to exploit multiple training options
    • Deep Neural Network:
      • Supported Theano or Tensorflow backend by Keras
      • CNN/FFNN, Convolution Neural Network/FeedForward Neural Network: for image or LIBSVM data
      • LSTM/RNN, Long Short-Term Memory/Recurrent Neural Network: for time series/sequential data
    • Spark MLlib:
      • Classification: Linear SVM and Logistic Regression with Grid Search and Cross Validation.
      • Clustering: Kmeans
    • SciKit-Learn:
      • Classification: Linear SVM, Logistic Regression, SVM, Passive Aggressive Classifier and Perception
      • Clustering: Kmeans
    • Ensemble classifier:
      • Allow selectable multiple classifiers from different datasets
      • Fast parallel processing
  • Adversarial Machine Learning Research:
    • Support pre-trained classifier/model for prediction: ImageNet/Inception or DarkNet/Yolo
    • Adversarial image generation by predefined universal perturbation filter
    • Fast processing: optimized by Intel KML or by GPU
  • Android Malware Research:
    • Dynamic study: link with Android Emulator Farm for APK execution and prediction
    • Static study: extract static data from APK for prediction
  • Data format option 1:
    • Input pre-featured data as "LIBSVM" format: (not for LSTM/RNN)
    • Each row for a sample
    • Format for a data row:<label> <index1>:<value1> <index2>:<value2> …\n
    • All data is numerical
    • Each feature represented by <indexN>:<valueN>
    • Meta Data: (optional)
      • Label list:<label name(string)>:<label(number)>
      • Feature list:<feature description(string)> : <indexN(number)>
  • Data format option 2:
    • Input raw data as "N-gram pattern" format with the Regular Expression pattern string specified
    • Each data row for a sample. Allow "time sequence" data. e.g. execution log etc
    • Each feature string for pattern matching separated by a "tab" char, "\t"
    • Format for a data row example:<label>\t<md5>\t<date>\t <string1 for pattern match>\t<string2 for pattern match>\t …\n
    • Pattern string example: '^I/AndroidATD\([ ]*[\d]+\): \[API\] (.*) \[.*'
      • substring(s) will be extracted as part of the feature in n-gram (e.g. highlighted in parenthesis)
    • Extracted string will be hashed and n-gram-ized to the "LIBSVM" format
    • Meta Data: (generated)
      • Label list:<label name(string)>:<label(number)>
      • Feature list:<indexN(number)> : <feature description(string)>
